# Melanie’s Smoothies

A Streamlit web application connected to Snowflake’s cloud data platform, built as part of the **Snowflake Badge 3: Data Application Builders Workshop**.

[![Snowflake Badge](https://img.shields.io/badge/Snowflake-Badge%203%3A%20Data%20App%20Builders-29B5E8?style=flat&logo=snowflake&logoColor=white)](https://achieve.snowflake.com/6a178323-8b69-4325-9241-3570562fde3e#acc.nabCfUFV)

-----

## What It Does

- Presents a web-based smoothie order form via Streamlit
- Reads fruit options and order data from Snowflake tables
- Writes new orders back to Snowflake in real time
- Integrates with the Smoothie Froot API to pull nutritional data per ingredient

## What I Learned

This project was hands-on practice connecting a Python front end to a cloud data warehouse. Key skills covered:

- Setting up Snowflake tables, stages, and warehouses
- Connecting Streamlit to Snowflake using the Snowflake Python connector
- Building a data application that reads and writes to a cloud database
- Consuming a third-party REST API and displaying results inline

## Tech Stack

|Tool              |Purpose                                         |
|------------------|------------------------------------------------|
|Snowflake         |Cloud data platform (tables, warehouse, compute)|
|Streamlit         |Python-based web front end                      |
|Python            |Application logic                               |
|Smoothie Froot API|Nutritional data per fruit ingredient           |
